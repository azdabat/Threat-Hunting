R&D / THREAT HUNTING PRACTICE — INTERNAL ROADMAP

THR‑ROADMAP: Coverage‑First Threat Hunting Programme

A phased hunt backlog built from the ATT&CK‑aligned hunt taxonomy, structured using PEAK (Prepare → Execute → Act with Knowledge), scoped with TaHiTI (Initiate → Hunt → Finalize), and hypothesised with the ABLE table (Actor, Behaviour, Location, Evidence). Ordered by realistic prevalence and impact across most environments — not by ATT&CK tactic order.

PEAK — Splunk SURGe TaHiTI — FI‑ISAC NL ABLE — Hypothesis Table MITRE ATT&CK v15+

00How the three frameworks fit together

PEAK governs the lifecycle of a single hunt. TaHiTI governs how the hunt programme as a whole is fed, prioritised, and closed out against other security processes. ABLE is the hypothesis‑construction tool used inside PEAK's Prepare phase (and TaHiTI's Define/Refine step) to turn a vague idea into something testable. This roadmap nests all three: TaHiTI sets the backlog and triggers, PEAK runs each hunt, ABLE shapes each hypothesis before a query is ever written.

PEAK hunt lifecycle

  1. Prepare — select topic, research, build ABLE hypothesis, scope data
  2. Execute — query, pivot, correlate, confirm/refute
  3. Act w/ Knowledge — escalate, document, convert to detection, re‑seed backlog

TaHiTI programme flow

  1. Initiate — trigger (CTI, prior hunt, monitoring gap, IR, exec ask) → abstract → backlog
  2. Hunt — define/refine hypothesis, execute, validate (proven / disproven / inconclusive)
  3. Finalize — document, handover to IR/detection engineering/vuln mgmt/CTI

ABLE hypothesis table

  1. Actor — who/what is doing it (insider, commodity loader, ransomware affiliate)
  2. Behaviour — the specific TTP, mapped to an ATT&CK technique ID
  3. Location — host type, log source, network segment, identity tier
  4. Evidence — the artefact/telemetry that would prove or disprove it
Worked ABLE example — used as the template for every hunt card below: Actor: post‑exploitation operator with local admin. Behaviour: LSASS memory access for credential material (T1003.001). Location: domain‑joined workstations and Tier‑0/1 servers, Sysmon Event ID 10 + EDR process‑access telemetry. Evidence: non‑allow‑listed process opening lsass.exe with PROCESS_VM_READ, comsvcs.dll MiniDump invocation, or unsigned binary handle requests.

01Roadmap structure & prioritisation logic

Hunts are grouped into four delivery phases. Phase 1 covers the techniques that show up across nearly every intrusion regardless of actor sophistication (commodity malware, ransomware affiliates, APT alike) — these are funded first because they close the widest detection gap per hour invested. Later phases add precision, automation maturity (M‑ATH/baseline hunts), and lower‑frequency‑but‑high‑impact techniques. Every phase still runs the full PEAK cycle and feeds TaHiTI's Finalize handover into detection engineering.
